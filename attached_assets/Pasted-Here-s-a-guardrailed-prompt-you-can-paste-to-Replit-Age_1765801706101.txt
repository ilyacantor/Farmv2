## Replit Agent Prompt: Farm Sanity Test Harness (Read-Only, Agent-Proof)

**ROLE:** You are implementing a *read-only* external sanity test harness for the **Farm** repo.
**GOAL:** Create a single command I can run anytime to detect “janky non-IRL modality” problems (dual storage, legacy snapshot folders, etc.) without modifying Farm behavior to make tests pass.

### Non-Negotiables

1. **DO NOT modify any existing Farm runtime code** (no changes to snapshot writers/readers, DB logic, pipelines, services, API routes).

   * You may only add a new script + optional minimal wiring to run it (e.g., Makefile entry or package script).
2. **The test must be read-only.**

   * No writes to DB. No creating/deleting files. No migrations. No “cleanup.” No “auto-fix.”
3. **The test must be hard to game.**

   * It must inspect the **filesystem + SQLite DB directly**, not via Farm’s internal “snapshot API” (which could lie).
4. **It must fail with exit code ≠ 0** if any guardrail is violated.
5. **It must be fast:** target runtime < 3 seconds.
6. **No scoring. No thresholds. Only binary pass/fail rules.**
7. **All checks and output must be deterministic and stable** across runs.

---

### What to Build

#### A) Create script

* Path: `tools/sanity/farm_sanity_check.py`
* It should print:

  * Header with timestamp and repo root
  * Each check name + PASS/FAIL
  * On failure: specific reason(s) and the exact path(s) / counts involved
  * Final line: `OVERALL: PASS` or `OVERALL: FAIL`
* Exit codes:

  * `0` = PASS
  * `2` = FAIL (guardrail broken)
  * `3` = ERROR (script couldn’t run due to missing paths/permissions)

#### B) Required checks (Phase 0)

1. **Dual storage detection**

   * Detect presence of BOTH:

     * `farm.db` (SQLite file) **and**
     * `data/snapshots/` (legacy JSON-on-disk folder)
   * This is only acceptable if **one is provably empty**.
   * Rules:

     * If `data/snapshots/` exists and contains any `.json` (or any files) → FAIL
     * If `farm.db` exists and is non-trivial (> 1MB) AND legacy folder has files → FAIL
     * If both exist but legacy folder empty → PASS (but print “legacy folder present but empty”)
2. **SQLite integrity + basic inventory**

   * If `farm.db` exists:

     * Run `PRAGMA integrity_check;` and FAIL if not `ok`
     * List table names
     * For the snapshot tables you can find (best-effort), print row counts
3. **Snapshot immutability hint**

   * Compute a hash summary:

     * `sha256(farm.db)` if present
     * plus `sha256` of directory listing metadata for `data/snapshots/` (names + sizes) if present
   * Print them so I can compare between runs. (Do NOT write a state file.)
4. **IRL-modality schema smell test (static)**

   * Scan the snapshot JSON files **if any exist** (but if any exist it’s already FAIL — still show a small sample of keys from up to 3 files to help diagnosis).
   * Scan DB column names (via PRAGMA table_info) and flag “synthetic-only” suspicious fields by name pattern:

     * `confidence`, `anomaly`, `score`, `synthetic`, `generated`, `mock`, `simulated`
   * If any are found, print warnings (NOT fail yet in Phase 0). We’ll decide fail rules later.

#### C) Add a single command I can run

Pick one and implement it:

* **Option 1 (preferred):** `make sanity`
* **Option 2:** `python tools/sanity/farm_sanity_check.py`

If you add a Makefile target, it must just call the python script.
