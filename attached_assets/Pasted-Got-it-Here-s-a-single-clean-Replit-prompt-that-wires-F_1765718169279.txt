## Replit Prompt — Wire Reconciliation into Farm UI (Plain-English, Run-by-Run)

You are extending the **Farm UI** to surface the existing **Reconciliation capability** in a clear, user-friendly way.

### Goal (plain English)

For any Farm snapshot and corresponding AOD run, a user should be able to:

* see **whether AOD agrees with Farm’s evidence**
* understand **what differs and why**, in plain English
* do this **without digging into raw JSON**

Reconciliation already exists server-side. This task is **UI wiring only**.

---

## UX Requirements (non-negotiable)

### 1. Reconciliation is snapshot-centric

* Reconciliation is always shown **in the context of a snapshot**
* Do NOT introduce a global “reconciliation dashboard”

User mental model:

> “For this snapshot, how did AOD do?”

---

### 2. Add a “Reconciliation” section to Snapshot Detail

In the Farm UI, wherever a snapshot is viewed (or listed):

Add a **Reconciliation panel** with:

* Status badge: **PASS / WARN / FAIL**
* AOD Run ID
* Timestamp
* One-sentence headline from `report_text`

If no reconciliation exists:

* Show a neutral empty state:

  > “No AOD reconciliation recorded for this snapshot yet.”

---

### 3. Simple action: “Reconcile with AOD run”

Add a button:

> **Reconcile with AOD run**

On click:

* Show a small modal or inline form:

  * Input: `AOD Run ID` (text)
  * Optional: paste or upload AOD summary JSON (if required by API)
* Submit calls `POST /api/reconcile`

No auto-discovery, no magic guessing. Explicit is fine.

---

### 4. Plain-English report rendering (most important)

Render the reconciliation report as **human text**, not JSON.

Display:

* Headline sentence (first line of `report_text`)
* Bullet list of:

  * “What matched”
  * “What differed”
  * “Most likely cause”
  * “What to do next”

Do **not** show raw expectation vs actual JSON unless user expands “Details”.

---

### 5. History (lightweight)

If multiple reconciliations exist for the same snapshot:

* Show a simple list (most recent first):

  * timestamp
  * status badge
  * AOD run id
* Clicking one loads its report

No charts. No timelines.

---

## UI Safety Rules

* Never assume reconciliation exists
* Never crash if fields are missing
* Treat `report_text` as trusted display text
* All network errors must show a visible inline error message

---

## API Wiring (assume backend already exists)

Use existing endpoints:

* `POST /api/reconcile`
* `GET /api/reconcile?snapshot_id=...`
* `GET /api/reconcile/{reconciliation_id}`

Do not change backend behavior.

---

## Visual Style

Match existing Farm UI:

* Clean, compact
* Status badges: green / yellow / red
* No dense tables
* Narrative > metrics

---

## Definition of Done

* User can open a snapshot and see reconciliation status (or lack thereof)
* User can trigger a reconciliation by entering an AOD run ID
* User can read a short, plain-English explanation of differences
* Multiple reconciliations per snapshot are viewable
* No raw JSON required to understand outcomes

---

## Deliverable

* Updated Farm UI with Reconciliation section
* One screenshot or short description showing:

  * a PASS or FAIL reconciliation rendered clearly
* No backend changes
