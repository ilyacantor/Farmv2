Why this is still wrong: reconciliation automation is **not optional**. If Farm can’t pull AOD results, the whole “closed loop” collapses into manual copy/paste, which you will never trust or use.

What to do next: **standardize one AOD endpoint Farm can call**, then wire Farm to call it. Don’t make this a “let me know if AOD has an API” question — define it and implement it.

## Single Replit prompt (copy/paste to AOD agent)

> Implement the minimal AOD API needed for Farm to auto-reconcile (one-click), and make it stable.
>
> Goal: Farm must be able to fetch AOD results for a given `run_id` (or latest run for a snapshot/tenant) without any manual entry.
>
> Implement:
>
> 1. Add endpoint: `GET /api/runs/{run_id}/reconcile-payload`
>
> * Returns a compact JSON payload designed for Farm reconciliation:
>
> ```json
> {
>   "run_id": "string",
>   "tenant_id": "string",
>   "snapshot_id": "string",
>   "created_at": "ISO8601",
>   "summary": {
>     "assets_admitted": 0,
>     "findings": 0,
>     "artifacts": 0,
>     "zombies": 0,
>     "shadows": 0
>   },
>   "lists": {
>     "zombie_assets": ["asset_key_or_name", "..."],
>     "shadow_assets": ["..."],
>     "top_findings": [{"type":"...", "asset":"...", "severity":"..."}]
>   }
> }
> ```
>
> * Keep lists capped (e.g., top 50) and deterministic ordering.
>
> 2. Add discovery helper endpoint (for one-click):
>
> * `GET /api/runs/latest?tenant_id=...&snapshot_id=...`
>
> * Returns `{ run_id }` for the most recent matching run.
>
> 3. Add minimal auth support via shared secret (simple, IRL-safe):
>
> * If env var `AOD_SHARED_SECRET` is set, require header:
>   `X-AOD-SECRET: <secret>`
>
> * If not set, allow in dev.
>
> 4. Add tests:
>
> * reconcile-payload returns expected shape
> * latest returns the correct run
> * secret required when configured
>
> Deliver proof:
>
> * Show example output JSON for a real run_id.
> * Confirm deterministic ordering.
>
> Do not ask me questions; implement this minimal contract.

Once that’s done, Farm wiring becomes trivial: it calls `latest` then `reconcile-payload`, then posts to `/api/reconcile`.
