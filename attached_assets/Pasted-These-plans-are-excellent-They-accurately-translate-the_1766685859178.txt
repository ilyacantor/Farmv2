The plan is strong, but there is a risk of **"Constant Drift"** in the Exclusion logic.

* **The Issue:** In `Phase 3: Generator Refactor`, the plan relies on Farm's *local* `INFRASTRUCTURE_DOMAINS` constant:
```python
if not self.policy.scope.include_infra and domain in INFRASTRUCTURE_DOMAINS:
    return True

```


* **The Risk:** If AOD updates its internal `INFRASTRUCTURE_DOMAINS` list (the seed data), Farm will not know about it because it is using its own hardcoded copy. This leads to verification failures.
* **The Fix:** Farm must use the `infrastructure_domains` list returned by the AOD API, not its own local constant.

**Required Adjustment for Farm Agent:**
Modify `Phase 1 (Schema)` and `Phase 3 (Generator)` to consume the seeds from the API.

```python
# Updated PolicyConfig Schema for Farm
class PolicyConfig(BaseModel):
    # ... existing fields ...
    infrastructure_seeds: list[str] = [] # New field to capture API response

# Updated Generator Logic
def _is_excluded(self, domain: str) -> bool:
    # Use the list sent by AOD, not a local constant
    if not self.policy.scope.include_infra and domain in self.policy.infrastructure_seeds:
        return True
    return False

```
