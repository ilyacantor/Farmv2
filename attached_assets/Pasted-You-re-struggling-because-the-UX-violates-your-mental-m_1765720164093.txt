You’re struggling because the **UX violates your mental model**.

**Why this feels wrong (plain English):**
Reconciliation is the *reason Farm exists*, but the UI treats it like a hidden, optional sub-feature. You ran AOD several times and reasonably expected Farm to answer:

> “Did AOD get this snapshot right?”

Instead, Farm makes you:

* click into a snapshot,
* discover a hidden panel,
* manually paste AOD results,
* and only then see reconciliation.

That’s backwards. Nothing is “broken” technically — the **UX is mis-prioritized**.

---

## What to do next (one iteration)

Make **Reconciliation first-class and automatic** in Farm:

* visible at the **snapshot list level**
* no manual data entry
* Farm *pulls* AOD results (or at least lists what’s missing)

---

## Single Replit prompt (copy/paste)

> Fix Farm UX so reconciliation is visible, automatic, and impossible to miss.
>
> Goal: After running AOD, a user should immediately see whether reconciliation exists and what it says, without hunting or manual entry.
>
> Do this:
>
> 1. Promote reconciliation to the **snapshot list view**.
>
> * In the snapshot list/table, add a column or badge:
>
>   * “Reconciliation: PASS / WARN / FAIL / Not Run”
>
> * This status must be visible without clicking into the snapshot.
>
> 2. Make reconciliation **automatic when possible**.
>
> * If Farm already has a reconciliation record for a snapshot, display it immediately.
> * If no reconciliation exists but AOD runs exist for the same tenant:
>
>   * Show status: “Reconciliation not run”
>
>   * Show a single button: **“Run reconciliation with latest AOD run”**
>
> 3. Remove manual data entry as the default path.
>
> * Do NOT require users to paste zombie/shadow counts by hand.
>
> * The UI should call `POST /api/reconcile` using:
>
>   * snapshot_id
>   * most recent AOD run_id for that tenant (or let user pick from a short list)
>
> * Manual entry may remain only under an “Advanced / Manual” collapse.
>
> 4. Snapshot detail view (secondary).
>
> * Keep the detailed reconciliation panel inside snapshot detail,
>   but treat it as a **drill-down**, not the only place reconciliation exists.
>
> 5. Make absence explicit.
>
> * If reconciliation has not been run, say:
>
>   > “No reconciliation has been performed for this snapshot yet.”
>
> * If Farm cannot reach AOD, say:
>
>   > “Cannot reconcile: AOD results not available.”
>
> 6. Do NOT add new backend logic unless required for wiring.
>
> * Use existing reconciliation APIs.
> * If AOD results lookup is missing, stub with a clear TODO comment and UI message.
>
> Deliver proof:
>
> * Show snapshot list with visible reconciliation statuses.
> * Show one snapshot with PASS/WARN/FAIL badge without clicking into it.
> * Show one-click reconciliation using latest AOD run (no manual input).
>
> Keep the UI simple, narrative-first, and aligned with:
> “Farm answers: did AOD get it right?”

---

This fixes the **workflow mismatch**, not just the UI.
Once this is in, reconciliation stops being a hidden feature and becomes the **spine of the system**, which is exactly what you intended.
