Replit prompt: add a 60-second “Nuke Prevention Check” + plain-English report

Implement a fast, repeatable “Nuke Prevention Check” that can be run on a schedule and outputs a plain-English PASS/FAIL summary.

Constraints:

Must finish in ~60 seconds locally.

Must not require manual input.

Must output a short plain-English report (no stack traces unless FAIL).

Must be runnable via one command: python scripts/nuke_check.py

Step 1 — Detect project

If this repo looks like Farm (has snapshot generator / /api/snapshots endpoints), run Farm checks.

If this repo looks like AOD (has /api/runs or pipeline), run AOD checks.

If ambiguous, print “UNKNOWN PROJECT” and exit non-zero.

Step 2 — Farm checks (if Farm)

Start app (or import and call generator directly if easier/faster).

Verify GET /api/snapshots returns JSON with items[] and each item has snapshot_id, tenant_id, created_at, schema_version.

Pick the most recent snapshot and GET /api/snapshots/{snapshot_id}:

response is valid JSON

has meta.schema_version == "farm.v1"

has planes.discovery.observations list

has required plane keys present (even if empty): idp, cmdb, cloud, endpoint, network, finance

Run a “no-cheat” scan over the snapshot JSON to ensure banned adjudication fields do NOT appear anywhere (e.g., is_shadow_it, inCMDB, rulesTriggered, conflictTypes, sourcePresence, parked_reason, ground_truth). If any found → FAIL.

Step 3 — AOD checks (if AOD)

Confirm FARM_URL env var exists; if missing → FAIL with plain English.

Call Farm list endpoint through AOD (preferred) or directly (if AOD proxy exists):

obtain one snapshot_id

Trigger a run using the normal Farm path (e.g., POST /api/runs/from-farm with tenant_id + snapshot_id).

Assert the run returns one of the explicit statuses:

UPSTREAM_ERROR / INVALID_SNAPSHOT / COMPLETED_NO_ASSETS / COMPLETED_WITH_RESULTS

If it returns “completed” without those semantics → FAIL.

Determinism mini-check:

Run the same snapshot twice back-to-back with same config.

Compare functional outputs (counts + sorted list of asset keys/names + finding types). They must match. If not → FAIL.

Step 4 — Plain English output format
At the end, print exactly:

NUKE CHECK: PASS (or FAIL)
Project: FARM or AOD
Timestamp: <iso>
Key results:

<bullet list of what was verified>

If FAIL:

What failed: <single sentence>

Likely cause: <single sentence>

What to do: <single sentence>

Step 5 — Add a tiny convenience runner

Add scripts/nuke_check.py

Add README_NUKE_CHECK.md with:

how to run it

suggested cadence: “daily” and “before any merge/deploy”

Deliver proof:

Show one successful run output in the console.
