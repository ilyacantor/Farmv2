Replit Prompt — Build “AOS Farm” from Scratch (IRL-First Synthetic Enterprise for AOD)

You are building AOS Farm from scratch. You have no prior context. Treat this as a net-new product inside the AutonomOS ecosystem.

Context in AOS

AutonomOS modules:

AOD (Discover): ingests raw enterprise signals and builds an asset catalog.

Farm (this project): generates IRL-plausible enterprise source-of-truth planes + raw observation streams so AOD can be tested like it’s operating in the wild.

Farm must not embed conclusions. Farm outputs raw evidence streams and independent planes, not “answers.”

What Farm Must Deliver

Farm must generate a single enterprise snapshot with multiple planes that represent a modern large enterprise:

Discovery Observations (what scanners/logs see)

IdP (Okta/Entra-like identity view)

CMDB (ServiceNow-like IT inventory)

Cloud Inventory (AWS/Azure/GCP-like resources)

Endpoint/MDM/EDR (device/app inventory)

Network/DNS/Proxy (network evidence)

Finance/Procurement (purchase/billing/expense evidence)

All planes must be independent (different IDs, different coverage, different naming).
Correlation between planes should be possible only via realistic keys: names, domains, hostnames, URIs, vendor/product strings.

Hard Rules (fail build if violated)

Do not output any fields that are “conclusions” (no shadow flags, no “in_cmdb”, no rules triggered, no “source_presence” booleans).

Do not output any shared join key across planes.

Do not output “ground truth” or labels for what AOD should conclude.

Farm must be deterministic by seed.

Realism Profiles (don’t do arbitrary missingness percentages)

You must implement realism via enterprise-specific mechanics, not random “10–20% missing”.

Implement three realism profiles:

clean

Mature governance org: CMDB/IdP coverage is strong, naming is consistent, finance is structured.

typical

Realistic enterprise: some drift, some aliasing, some stale owners, multiple naming conventions, occasional duplicates.

messy

Post-M&A / decentralized: overlapping tools, duplicate app families, ambiguous names, inconsistent domains, multiple similar CIs.

Make realism differences come from:

name drift rules (prefix/suffix/abbrev)

alias collisions (e.g., “Salesforce” family)

staleness (old owners/emails)

duplicates and near-duplicates

inconsistent external_ref formats

Not from crude random field deletion targets.
