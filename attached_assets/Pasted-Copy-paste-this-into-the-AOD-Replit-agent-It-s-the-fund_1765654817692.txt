# Non-Negotiables (Fail the task if any violated)

1. **No “Run with Sample Data” button.**
   Delete UI, endpoints, and code paths that run without fetching from Farm.

2. **Farm URL is configured only via environment.**

* Read from `FARM_URL` env var (required).
* UI must not allow the user to enter Farm URL.

3. **Snapshot selection is not free-text only.**

* UI must fetch the list of snapshots from Farm and present a dropdown.
* Allow manual Snapshot ID entry only as an “Advanced” collapsed section.

4. **IRL status semantics.**
   AOD must never present “0 assets, 0 findings” as a successful run unless it truly completed.
   AOD must classify and show these distinct outcomes:

* `UPSTREAM_ERROR` (Farm unreachable / non-JSON / HTTP error)
* `INVALID_SNAPSHOT` (schema mismatch / wrong version / missing planes)
* `COMPLETED_NO_ASSETS` (pipeline completed; nothing admitted)
* `COMPLETED_WITH_RESULTS` (normal success)

5. **Strict input contract.**
   Snapshot must contain `meta.schema_version == "farm.v1"`. Reject others.

6. **Explainable, deterministic.**
   No anomaly scores. No ML. Same snapshot → same outputs.

# AOD API Changes (Backend)

## 1) Add/ensure endpoint: Run from Farm

`POST /api/runs/from-farm`

Request:

```json
{
  "tenant_id": "string",
  "snapshot_id": "string"
}
```

Behavior:

* Server reads `FARM_URL` from env
* Fetches snapshot from Farm
* Validates:

  * HTTP status code 2xx
  * `Content-Type` includes `application/json`
  * Body non-empty and valid JSON
  * `meta.schema_version == "farm.v1"`
  * Required plane keys exist
* If fetch fails → `UPSTREAM_ERROR` with details
* If validation fails → `INVALID_SNAPSHOT` with details
* Otherwise run pipeline, persist run, return summary with one of:

  * `COMPLETED_NO_ASSETS`
  * `COMPLETED_WITH_RESULTS`

Response:

```json
{
  "run_id": "string",
  "status": "UPSTREAM_ERROR|INVALID_SNAPSHOT|COMPLETED_NO_ASSETS|COMPLETED_WITH_RESULTS",
  "summary": {
    "observations_in": 0,
    "candidates_out": 0,
    "assets_admitted": 0,
    "artifacts_recorded": 0,
    "findings": 0,
    "ambiguous_matches": 0
  },
  "error": { "code": "string", "message": "string", "details": {} }
}
```

## 2) Add endpoint: List snapshots (proxy for UI)

To simplify the UI and avoid CORS complexity, AOD should proxy Farm snapshot listing:

`GET /api/farm/snapshots?tenant_id=...`

Server calls Farm list endpoint using `FARM_URL`, returns same metadata list.
If Farm fails → return `UPSTREAM_ERROR` shape.

## 3) Persist run provenance

Store on each run:

* `tenant_id`
* `farm_url_used` (from env)
* `snapshot_id`
* `schema_version`
* fetch timing + status

This is required for IRL auditability.
