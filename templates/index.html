Agent Stress Testing Guide

Learn how to generate test data for your AI agent platform

What is this?

This tool generates realistic test data for AI agent orchestration platforms. Instead of building fake test cases by hand, you can create hundreds of synthetic agents and workflows with a single click. The generated data includes built-in "chaos" events (simulated failures) to test how well your platform handles real-world problems.

1

Agent Fleet

Creates a team of AI agents with different roles: Planners break down complex tasks, Workers execute them, Specialists handle specific domains, and Reviewers/Approvers validate work.

2

Workflows

Creates task graphs that agents need to complete. Linear = step-by-step, DAG = complex dependencies, Parallel = many tasks at once, Saga = tasks with rollback capability.

3

Stress Scenario

Combines agents + workflows into a complete test package. Includes expected results so you can automatically verify if your platform performed correctly. Great for integration testing.

4

Run Against Platform

The power feature: sends test data directly to your platform, waits for results, and validates them automatically. Shows you exactly which checks passed or failed.

Quick Start

  1. 1. Just exploring? Click "Generate Fleet" or "Generate Stress Scenario" to see example data.
  2. 2. Testing your platform? Use "Run Against Platform" - enter your platform's URL and click run.
  3. 3. Building an integration? See the technical docs at /docs/agent-orchestration-guide.md

What is "Chaos"?

Chaos is intentional failures injected into workflows. Your platform should handle these gracefully - retrying failed tasks, triggering compensations, or escalating to humans. The chaos rate controls how many tasks will have problems. Start with 10-20% for realistic testing.
